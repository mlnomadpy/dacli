# adopter.md is a code-grounded composite persona tagged unverified, not a transcript of a real person
## Chose
adopter.md is a code-grounded composite persona tagged unverified, not a transcript of a real person
## Rejected
Inventing a named real interviewee, or leaving the file as an empty guide-shaped template
## Because
INTERVIEW_GUIDE.md §9 sets the trust floor: a single source is 'unverified — a lead, not a fact'. No real adopter has been interviewed yet (docs/research/transcripts/ is empty), so fabricating a real subject would launder fiction as recorded evidence. A composite persona whose every reaction is anchored to real dacli behavior (init output wscore.go:87-106, read-only dashboard dashboard.go:66-88, --yolo removing the between-cycle checkpoint orchestration.go:136-137/358-362) is an honest research artifact: it stands in until live interviews run, is explicitly labelled unverified, and answers §4 fully without pretending to be data it isn't.
