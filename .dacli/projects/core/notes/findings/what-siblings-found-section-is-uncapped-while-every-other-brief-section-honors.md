# What-siblings-found section is uncapped while every other brief section honors MillerCap
brief.go Assemble(): decisions (brief.go:128), risks (:160), and lessons (:195) each stop at MillerCap=7 with an announced omission, per the stated 'a brief is a working-memory budget, not an archive' contract (brief.go:45-48). The 'What siblings found' loop (brief.go:222-237) has no such cap — it renders every finding note in the project. With the default budget 0 (unlimited, briefing.go:32) nothing is trimmed, so a project with dozens of findings dumps all of them into the brief (this very task's brief shows 40+). If findings are meant to be uncapped by design, say so; otherwise apply MillerCap with an announced omission like the sibling sections.
